# InkDesk 남은 UX/기능 수정 요청

## 현재 상태 요약

Claude 수정 후 InkDesk는 범용 소상공인 상담툴에서 **타투이스트/타투 스튜디오 전용 상담 CRM**으로 꽤 많이 좁혀졌다.

확인된 개선 사항은 다음과 같다.

- 기본 업종이 `tattoo`로 고정됨
- 브랜드가 `InkDesk`, `타투 상담 자동화`로 정리됨
- 카테고리가 `견적 문의`, `예약/일정`, `커버업`, `리터치`, `관리/애프터케어`로 변경됨
- 상담 접수 폼에 `시술 부위`, `크기`, `스타일`, `커버업 여부`가 추가됨
- 고객 관리에 `피부 특이사항`이 추가됨
- 권한 구조가 `대표/팀원` 2단계로 단순화됨
- 샘플 문의와 기본 지식베이스가 타투 상담 기준으로 변경됨
- DB 스키마에 `tattoo_area`, `tattoo_size`, `tattoo_style`, `is_coverup`, `session_count`, `quoted_price` 컬럼이 추가됨
- E2E smoke test가 현재 정책 기준으로 수정되어 통과함

검증 결과:

```text
npm run typecheck  통과
npm run test:e2e   24 checks 통과
```

이 문서는 이미 해결된 항목을 제외하고, **다음 수정에서 집중해야 할 남은 문제**만 정리한다.

## 제품 기준 문장

다음 수정의 기준은 아래 문장으로 잡는다.

```text
InkDesk는 타투이스트가 DM/카톡 문의를 놓치지 않고,
필요한 상담 정보를 빠르게 확인한 뒤,
견적과 예약금 안내까지 자연스럽게 보내게 해주는 상담 CRM이다.
```

현재 제품은 타투 전용화의 출발점은 좋지만, 아직 핵심 흐름이 `문의 처리` 중심에 가깝다. 다음 단계에서는 **상담을 예약으로 전환하는 과정**이 화면과 상태에 더 명확하게 드러나야 한다.

## 1. 상태값을 타투 예약 전환 파이프라인으로 변경

### 현재 문제

현재 상태값은 아직 일반 CS툴에 가까운 구조다.

```text
새 문의
초안 완료
보류
확인 필요
처리 완료
```

타투 상담에서는 문의가 "처리 완료"되는 것보다 **예약으로 전환되고, 시술까지 이어졌는지**가 더 중요하다.

### 수정 방향

상태값을 아래 흐름으로 바꾸는 것이 좋다.

```text
신규 문의
정보 요청
견적 안내
예약금 대기
예약 확정
시술 완료
리터치/관리
상담 종료
```

### 세부 수정 제안

- `Status` 타입 변경
- `statusLabels` 변경
- 문의 리스트 필터 변경
- 상세 패널의 상태 드롭다운 변경
- 사이드바/오늘 요약 지표 변경
- 리포트 화면의 상태 집계 변경
- E2E 테스트의 상태값도 새 정책에 맞게 수정

### 권장 매핑

기존 데이터 호환을 위해 일단 아래처럼 매핑하면 된다.

```text
new       → 신규 문의
drafted   → 견적 안내
pending   → 정보 요청
escalated → 예약금 대기 또는 확인 필요
done      → 상담 종료
```

다만 최종 제품에서는 영어 키도 의미에 맞게 바꾸는 편이 좋다.

```text
new
info_requested
quoted
deposit_pending
booked
completed
aftercare
closed
```

## 2. 문의 상세 화면에 "부족한 정보"와 "다음 질문" 추가

### 현재 문제

상담 접수 폼에 타투 필드는 생겼지만, 문의 상세 화면은 아직 답변 초안 중심이다.

타투이스트가 가장 먼저 알고 싶은 것은 다음이다.

```text
이 고객에게 견적을 내기 위해 무엇이 부족하지?
내가 다음에 뭐라고 물어봐야 하지?
예약금 안내까지 보내도 되는 상태인가?
```

### 수정 방향

문의 상세 상단 또는 추천 답변 카드 위에 **상담 체크리스트**를 추가한다.

### 추가 UI 예시

```text
상담 정보 체크

확인됨
- 시술 부위: 손목
- 크기: 5cm
- 스타일: 레터링

부족함
- 참고 이미지
- 희망 날짜
- 컬러 여부

다음 질문
"견적을 더 정확히 안내드리기 위해 참고 이미지와 희망 날짜를 보내주실 수 있을까요?"
```

### 구현 제안

- `Inquiry`의 현재 타투 필드 기준으로 부족한 항목 계산
- 메시지 텍스트에서 일부 정보 자동 추출 가능하면 더 좋음
  - `5cm`, `10x10`, `손목`, `팔`, `등`, `레터링`, `커버업` 등
- 처음에는 완전 자동 추출보다 UI 체크리스트 표시만 해도 충분함
- 부족한 정보에 따라 추천 질문 문장을 규칙 기반으로 생성
- `추천 답변`에는 이 질문이 자연스럽게 반영되도록 연결

## 3. AI 초안 메뉴를 상담 상세 흐름에 통합

### 현재 문제

`AI 초안` 메뉴가 아직 별도 네비게이션으로 남아 있다.

사용자는 문의를 선택한 뒤 바로 답변을 만들고 싶어 한다. 별도 메뉴로 이동해서 AI 초안을 만드는 흐름은 실제 업무 동선과 맞지 않는다.

### 수정 방향

AI 기능은 `상담 처리` 화면의 상세 패널 안에 통합한다.

### 세부 수정 제안

- 사이드바에서 `AI 초안` 메뉴 제거 또는 하위 기능으로 축소
- 상세 패널 안에 버튼 배치
  - `답변 초안 다시 생성`
  - `추가 질문 만들기`
  - `예약금 안내 포함`
  - `애프터케어 안내 포함`
- AI 결과는 현재 답변 편집 영역에 바로 반영
- AI 실행 전 지식베이스 부족 상태 안내

### 안내 문구 예시

```text
예약금 정책이 비어 있어 예약 확정 답변이 일반적으로 작성됩니다.
예약금/취소 정책을 추가하면 답변 품질이 좋아집니다.
```

## 4. 지식베이스를 타투 상담 섹션으로 세분화

### 현재 문제

현재 지식베이스는 다음 두 축으로 되어 있다.

```text
가격/서비스 항목
자주 쓰는 안내 문구
```

이 구조는 이전보다 낫지만, 타투 상담에는 아직 너무 넓다. 타투 상담 답변 품질에 중요한 기준들이 한 덩어리로 섞인다.

### 수정 방향

지식베이스를 타투 상담에 맞는 섹션으로 나눈다.

```text
견적 기준
예약금/취소 정책
작업 가능/불가 조건
커버업 안내
시술 전 주의사항
시술 후 관리법
리터치 정책
```

### 세부 수정 제안

- `가격/서비스 항목` → `견적 기준`
- `자주 쓰는 안내 문구` → 여러 정책 섹션으로 분리
- 각 섹션에 기본 예시 제공
- 답변 미리보기에서 어떤 섹션이 반영되는지 표시
- AI 초안 생성 시 관련 섹션만 컨텍스트로 넘길 수 있도록 구조화

### 현실적인 1차 구현안

DB 구조를 크게 바꾸기 부담스럽다면, 우선 기존 `prices`, `faq` 저장 구조를 유지하되 UI만 아래처럼 분리해도 된다.

```text
prices:
- 견적 기준

faq:
- 예약금/취소 정책
- 작업 가능/불가 조건
- 커버업 안내
- 시술 전 주의사항
- 시술 후 관리법
- 리터치 정책
```

장기적으로는 `business_knowledge`를 JSON 구조로 바꾸는 것이 좋다.

## 5. 오늘 요약/리포트를 타투 운영 지표로 변경

### 현재 문제

사이드바와 대시보드 지표가 아직 일반 문의 처리 지표에 가깝다.

```text
미처리
긴급
처리 완료
답변 준비
```

타투이스트에게 더 중요한 것은 상담이 예약으로 넘어가는 흐름이다.

### 수정 방향

오늘 요약과 리포트를 아래 지표 중심으로 바꾼다.

```text
신규 상담
정보 부족
견적 안내
예약금 대기
예약 확정
오늘 시술
리터치/관리 문의
```

### 세부 수정 제안

- 사이드바 `오늘 요약` 수정
- 문의 화면 상단 metric 수정
- 리포트 화면의 카드/차트 문구 수정
- 기존 `urgent`는 완전히 제거하지 말고 보조 배지로 유지

## 6. 온보딩을 더 타투 스튜디오 기준으로 다듬기

### 현재 개선된 점

업종 선택 UI는 사실상 사라졌고, `InkDesk는 타투 스튜디오 전용입니다`라는 안내가 들어간 점은 좋다.

### 아직 아쉬운 점

온보딩 단계 이름이 아직 다음처럼 범용 설정 느낌이 남아 있다.

```text
업종
접수
지식
팀
확인
```

### 수정 방향

단계 이름을 실제 타투 운영 흐름에 맞게 바꾼다.

```text
응대 기준
상담 접수
견적 기준
예약 정책
확인
```

또는 더 명확하게:

```text
스튜디오
상담 항목
견적/예약금
안내 문구
팀
```

### 세부 수정 제안

- `Workspace onboarding` → `Studio onboarding`
- `처음 운영 세팅` → `스튜디오 상담 기준 만들기`
- `가격표와 FAQ` → `견적 기준과 예약 정책`
- 완료 후 안내 문구:

```text
상담 화면에서 실제 DM/카톡 문의를 넣어보고,
답변 소스에서 예약금·애프터케어 문구를 스튜디오 기준으로 조정하세요.
```

```text
문의 접수 → 상담 정보 확인 → 견적 안내 → 예약금 대기 → 예약 확정 → 시술/리터치 관리
```

- 현재 구현된 기능
- 아직 미구현인 기능

## 8. 데이터/DB 주의사항

### 이미 해결된 문제

`tattoo_area` 등 타투 상담 필드는 스키마 파일에 추가되어 있고, 로컬 DB에는 `npm run db:schema`를 실행해 반영했다.

### 다음 수정 시 주의할 점

상태값이나 지식베이스 구조를 바꾸면 DB와 E2E도 같이 수정해야 한다.

수정 후 반드시 아래를 실행한다.

```powershell
npm run db:schema
npm run typecheck
npm run test:e2e
```

특히 다음 변경은 API, DB, 테스트가 같이 움직여야 한다.

- `Status` 타입 변경
- `statusLabels` 변경
- 리포트 상태 집계 변경
- `business_knowledge` 구조 변경
- 문의 저장/수정 API 필드 추가
- 고객 상태값 변경

## 우선순위

다음 수정은 아래 순서가 좋다.

1. 상태값을 타투 예약 전환 파이프라인으로 변경
2. 문의 상세 화면에 부족한 정보/다음 질문 카드 추가
3. AI 초안을 문의 상세 화면에 통합하고 별도 메뉴 축소
4. 지식베이스를 타투 정책 섹션으로 세분화
5. 오늘 요약/리포트를 예약 전환 지표로 변경
6. 온보딩 문구와 단계명을 스튜디오 기준으로 정리
7. README를 InkDesk 기준으로 업데이트

## 최종 판단

현재 InkDesk는 타투이스트 특화 MVP로 전환되는 방향이 맞고, 기본 동작도 검증됐다.

다음 수정의 핵심은 기능을 더 많이 넣는 것이 아니라, 화면의 중심을 아래처럼 바꾸는 것이다.

```text
문의 처리 도구
↓
타투 상담을 예약으로 전환하는 업무 흐름 도구
```
