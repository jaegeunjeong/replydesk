# ReplyDesk Lite — Frontend Design Brief

## 프로젝트 개요

소상공인(미용실, 학원, 공방, 병원, 정비소)이 고객 문의를 한 곳에서 접수·분류하고, AI 답변 초안·고객 이력·지식베이스·팀 권한·일일 리포트까지 관리하는 **상담 운영 SaaS 대시보드**입니다.

- **기술 스택**: Next.js 15 (App Router) + React 19 + TypeScript
- **언어**: 전체 UI가 한국어, eyebrow 라벨만 영어
- **타겟**: 직원 1~10명 규모의 한국 소상공인 매장

## 전체 레이아웃 구조

### App Shell
```
┌─────────────┬──────────────────────────────────────┐
│  Sidebar    │  Workspace (main content)            │
│  272px      │  - Topbar (sticky)                   │
│  sticky     │  - View-specific content             │
│  100vh      │                                      │
│             │                                      │
│  - Brand    │                                      │
│  - Nav      │                                      │
│  - Summary  │                                      │
└─────────────┴──────────────────────────────────────┘
```

- 사이드바: 다크 배경, 고정 좌측, 브랜드 로고 + 네비게이션 + 오늘 요약
- 메인 영역: 라이트 배경, 상단 고정 탑바 + 스크롤되는 콘텐츠
- 반응형: 980px 이하에서 단일 컬럼

### 인증 안 된 상태
- 전체 화면 로그인/회원가입 폼 (단일 패널, 중앙 정렬)

```
┌─────────────────────────────────────────────────────────────┐
```

```
│  카카오톡, DM, 문자 문의를 한 번에 정리합니다.                │

```

#### 4-3. 문의 리스트 + 상세 (2열 레이아웃)

```
┌──────────────────────────┬──────────────────────────────────┐
│  처리 대기  [Queue]      │  [Selected inquiry]              │
│  [상태 ▼] [유형 ▼]      │  김민지                          │
│  [고객 ▼]               │  내일 오후 3시에 예약 가능한가요? │

│  ┌────────────────────┐  │                                  │
│  │ 김민지       [▼상태]│  │  ─── 추천 답변 ───              │
│  │ 내일 오후 3시에     │  │  ┌──────────────────────────┐   │
│  │ 예약 가능한가요?    │  │  │ textarea (답변 편집)      │   │
│  │ [예약] [보통] [카톡]│  │  └──────────────────────────┘   │
│  └────────────────────┘  │  [답변 복사] [수정 저장] [완료]  │
│                          │                                  │
│  ┌────────────────────┐  │  ─── 다음 행동 ───              │
│  │ 박서연       [▼상태]│  │  답변을 복사해서... [문의 삭제]  │
│  │ 가격이 어떻게      │  │                                  │
│  │ 되나요?            │  │  ▸ 처리 정보 (접이식, 기본 열림) │
│  │ [가격] [긴급] [전화]│  │    상태/우선순위/담당자 select   │
│  └────────────────────┘  │    내부 메모 textarea            │
│                          │                                  │
│  ...                     │  ▸ 분류와 품질 (접이식)          │
│                          │    채널/유형/담당자/톤 메타       │
│                          │    품질점수/수정량/금지표현       │
│                          │    AI원본 vs 최종답변 비교       │
│                          │                                  │
│                          │  ▸ 이력 (접이식)                │
│                          │    타임라인 리스트               │
│                          │    수정 이력 리스트              │
│                          │    같은 고객 이전 문의           │
└──────────────────────────┴──────────────────────────────────┘
```

**문의 카드 구조:**
- 상단: 고객명 + 상태 select 드롭다운
- 중간: 문의 메시지 텍스트
- 하단: [유형 뱃지] [우선순위 뱃지] + [채널 뱃지]
- 선택 시: primary 테두리 + glow shadow
- hover: translateY(-1px) + 그림자 확장

**상세 패널 구조:**
- reply-hero: 고객명 + 메시지 + 상태 뱃지들
- reply-card: 답변 편집 textarea (240px 높이) + 액션 버튼 3개
- next-step-card: 다음 행동 안내 + 삭제 버튼
- support-drawer: 3개의 접이식 details/summary 섹션
  - 처리 정보: 3열 그리드 (상태/우선순위/담당자) + 내부 메모
  - 분류와 품질: 메타 그리드 + 품질 그리드 + 경고 뱃지 + AI 비교
  - 이력: 타임라인 + 수정 로그 + 고객 과거 문의

## 인터랙션 패턴

### 버튼
- primary: teal 배경, 흰 텍스트, teal glow shadow
- secondary: 흰 배경, 회색 테두리
- danger: 연빨강 배경, 빨간 텍스트, 빨간 테두리
- 모든 버튼: hover 시 translateY(-1px) + shadow 확장
- disabled: opacity 0.5, pointer-events none

### 카드/항목
- hover: translateY(-1px) + 테두리 강화 + shadow 확장
- selected: primary 테두리 + primary glow (0 0 0 3px rgba)

### 입력 필드
- focus: primary 테두리 + primary glow ring (3px)
- 체크박스 선택 시: primary 테두리 + primary-surface 배경

### 모달
- 배경: 어두운 반투명 + backdrop-filter blur
- 등장: fadeIn + slideUp 애니메이션

### 접이식 (details/summary)
- summary hover: 배경색 전환
- +/- 아이콘으로 열림/닫힘 표시
